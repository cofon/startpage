# StartPage Edge 浏览器插件

这是一个 Edge 浏览器插件，用于快速添加网站到起始页，并支持数据的导入导出。

## 📦 功能特性

### 1. 快捷键添加网站
- **快捷键**: `Alt + Shift + D` (Windows) / `Alt + Shift + D` (Mac)
- 自动填充当前网页的标题和 URL
- 可自定义网站名称、描述、标签等信息
- 支持标记为"常用网站"（显示在起始页）

### 2. 数据导入
- 支持从现有起始页项目导入备份的 JSON 数据
- 批量导入所有网站数据
- 自动处理数据格式兼容性

### 3. 数据导出
- 导出所有网站数据到 JSON 文件
- 可用于数据备份或迁移到其他设备
- 文件名自动生成时间戳

## 🚀 安装方法

### 方法一：开发者模式安装（推荐）

1. **打开 Edge 扩展管理页面**
   - 地址栏输入：`edge://extensions/`
   - 或点击菜单 → 扩展 → 管理扩展

2. **启用开发者模式**
   - 打开左侧的"开发人员"开关

3. **加载扩展**
   - 点击"加载解压缩的扩展"
   - 选择 `d:\vue\startpage\extension` 文件夹
   - 确认加载成功

4. **验证安装**
   - 在扩展列表中看到 "StartPage - 快速添加网站"
   - 版本号为 `1.0.0`

### 方法二：打包安装

1. 在扩展管理页面点击"打包扩展"
2. 选择扩展目录：`d:\vue\startpage\extension`
3. 生成 `.crx` 和 `.zip` 文件
4. 双击 `.crx` 文件安装

## 📖 使用指南

### 添加单个网站

1. **使用快捷键**
   - 在任何网页上按下 `Alt + Shift + D`
   - 弹出插件窗口

2. **填写信息**
   - 网站名称（必填）：例如 "Google"
   - 网站 URL（必填）：例如 "https://www.google.com"
   - 描述（可选）：网站的简短描述
   - 标签（可选）：用逗号分隔，例如 "搜索，工具"
   - 勾选"标记为常用网站"可在起始页显示

3. **保存**
   - 点击"保存"按钮
   - 看到"✓ 添加成功"提示

### 导入现有数据

1. **准备数据文件**
   - 从现有起始页项目导出数据
   - 获取 JSON 格式的备份文件（例如：`startpage-backup-2026-03-10.json`）

2. **打开导入面板**
   - 点击插件图标打开弹窗
   - 切换到"导入数据"标签页

3. **选择文件**
   - 点击"选择 JSON 文件"
   - 选择你的备份文件
   - 文件名会显示在下方

4. **开始导入**
   - 点击"开始导入"按钮
   - 等待进度条完成
   - 查看导入结果（成功导入 X 个网站）

5. **验证数据**
   - 切换到"添加网站"标签页
   - 或在起始页查看已导入的网站

### 导出数据

1. **打开导出面板**
   - 点击插件图标
   - 切换到"导出数据"标签页

2. **导出数据**
   - 点击"导出数据"按钮
   - 自动下载 JSON 文件
   - 文件名格式：`startpage-backup-YYYY-MM-DD-HH-mm-ss.json`

3. **保存备份**
   - 将导出的文件保存到安全位置
   - 可用于恢复或迁移到新设备

## 🗄️ 数据存储

### 数据库结构

- **存储位置**: 浏览器 IndexedDB 数据库
- **数据库名称**: `StartPageDB`
- **表名**: `websites`
- **主键**: `id` (自动递增)

### 数据字段

| 字段名 | 类型 | 必填 | 说明 | 使用场景 |
|-------|------|------|------|---------|
| `id` | Number | 自动生成 | 唯一标识符（自增） | 内部使用 |
| `name` | String | ✅ | 网站名称（用户自定义） | **Marked List 显示** |
| `title` | String | ✅ | 网站标题（网页原始标题） | **搜索结果显示** |
| `url` | String | ✅ | 网站 URL | 链接跳转 |
| `description` | String | 可选 | 网站描述 | 辅助信息 |
| `tags` | Array<String> | 可选 | 标签数组 | 搜索/分类 |
| `isMarked` | Boolean | 可选 | 是否标记为常用 | 过滤控制 |
| `markOrder` | Number | 可选 | 标记顺序（用于排序） | Marked List 排序 |
| `isActive` | Boolean | 可选 | 是否激活 | 过滤控制 |
| `isHidden` | Boolean | 可选 | 是否隐藏 | 过滤控制 |
| `visitCount` | Number | 可选 | 访问计数 | 统计信息 |

### 字段说明

#### **name 字段 - 用户自定义名称**
- **来源**：用户在添加/编辑网站时手动输入
- **用途**：在 **Marked List（已标记网站列表）** 中显示
- **特点**：简短、精炼，适合快速浏览
- **示例**："百度"、"GitHub"、"B 站"

#### **title 字段 - 网页原始标题**
- **来源**：从网页的 `<title>` 标签自动获取
- **用途**：在 **搜索结果列表** 中显示
- **特点**：完整、准确，反映网页实际内容
- **示例**："百度一下，你就知道"、"GitHub: Let's build from here"

#### **使用场景区分**

```
┌─────────────────────────────────────┐
│  Marked List 模式（显示 name）       │
├─────────────────────────────────────┤
│  [百度] [GitHub] [B 站] [YouTube]   │
│   ↑ 显示 name 字段                   │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  搜索结果模式（显示 title）          │
├─────────────────────────────────────┤
│  🔍 百度一下，你就知道               │
│     https://www.baidu.com           │
│     ↑ 显示 title 字段                │
│                                     │
│  🔍 GitHub: Let's build from here   │
│     https://github.com              │
│     ↑ 显示 title 字段                │
└─────────────────────────────────────┘
```

### 导入导出数据格式

**导入数据示例**（包含独立的 name 和 title）：

```json
{
  "websites": [
    {
      "id": 1,
      "name": "百度",                    // 用户自定义名称（用于 Marked List）
      "title": "百度一下，你就知道",      // 网页原始标题（用于搜索结果）
      "url": "https://www.baidu.com/",
      "description": "全球最大的中文搜索引擎",
      "tags": ["搜索", "search"],
      "isMarked": true,
      "markOrder": 1,
      "isActive": true,
      "isHidden": false,
      "visitCount": 0
    },
    {
      "id": 2,
      "name": "GitHub",                  // 用户自定义名称
      "title": "GitHub: Let's build from here",  // 网页原始标题
      "url": "https://github.com",
      "description": "代码托管平台",
      "tags": ["开发", "代码"],
      "isMarked": true,
      "markOrder": 2,
      "isActive": true,
      "isHidden": false,
      "visitCount": 0
    }
  ]
}
```

**导出数据格式**（保持原样导出）：

导出的 JSON 文件会完整保留所有字段信息，包括 `name` 和 `title`：

``json
{
  "websites": [
    {
      "id": 1,
      "name": "百度",
      "title": "百度一下，你就知道",
      "url": "https://www.baidu.com/",
      "description": "",
      "tags": ["搜索", "search"],
      "isMarked": true,
      "isActive": true,
      "isHidden": false,
      "visitCount": 0
    }
  ]
}
```

### 字段独立性原则

**重要**：`name` 和 `title` 是两个完全独立的字段：

1. ✅ **不会互相映射**：导入时保持原样，不自动转换
2. ✅ **不会互相覆盖**：各自独立存在，互不影响
3. ✅ **按需使用**：不同场景读取不同的字段
4. ✅ **可单独存在**：可以只有 name 或只有 title

### 存储内容说明

- ✅ **包含**：网站名称（name）、标题（title）、URL、描述、标签、标记状态等元数据
- ❌ **不包含**：图标数据（由起始页动态获取并缓存）

## 🔄 与起始页集成

插件和起始页共享同一个 IndexedDB 数据库：

1. **数据同步**
   - 插件添加的网站会立即出现在起始页
   - 起始页的操作也会反映到插件中
   - 无需手动刷新或同步

2. **使用方法**
   - 将起始页设置为浏览器主页或新标签页
   - 地址：`file:///d:/vue/startpage/dist/index.html`
   - 或使用本地服务器：`http://localhost:5173`

## ⚠️ 注意事项

1. **数据备份**
   - 定期导出数据以防丢失
   - IndexedDB 数据存储在本地，清除浏览器数据会删除

2. **跨浏览器**
   - 不同浏览器的 IndexedDB 不互通
   - 可通过导入导出功能迁移数据

3. **隐私模式**
   - 隐私/无痕模式下 IndexedDB 会被清除
   - 不建议在隐私模式下使用

4. **数据量限制**
   - IndexedDB 通常有存储空间限制（因浏览器而异）
   - 建议保持合理数量的网站数据

## 🐛 故障排除

### 问题 1：快捷键不生效

**解决方案**：
- 检查是否有其他扩展占用了相同快捷键
- 在 `edge://extensions/shortcuts` 中重新设置

### 问题 2：导入失败

**可能原因**：
- JSON 文件格式不正确
- 数据缺少必需的字段

**解决方案**：
- 确保 JSON 文件包含 `websites` 数组
- 检查每个网站对象是否有 `name` 和 `url` 字段

### 问题 3：数据不显示

**解决方案**：
- 刷新起始页
- 清除浏览器缓存后重试
- 检查 IndexedDB 中是否有数据
