

### 扩展描述

- 在任意一个浏览器标签页按下快捷键 alt+shift+d 打开扩展的添加网站面板
- 面板上显示以下内容
    - title输入框(input)
    - description输入框(input)
    - icon预览(img)
    - iconData输入框(textarea)
    - 取消按钮(button)
    - 添加按钮(button)
- 打开面板之后自动填充当前网页的数据到面板的输入框中
    - title 填充当前网页的标题
    - description 填充当前网页的描述
    - icon预览显示 iconData 的 icon
    - iconData 填充当前网页的icon转base64数据
- 添加按钮点击之后, 添加网站
    - 如果有起始页实例，发送消息给起始页，让起始页添加网站
    - 如果没有起始页实例，添加数据到扩展的仓库中
- 取消按钮点击之后，关闭面板


### 扩展仓库

- 扩展仓库保存的数据有
    - metas : [meta]
    - 其他数据(你可以根据需要添加，如果不需要，就只保存 meta)
    - 已经同步的数据删除
- meta 数据结构为
    - url: string
    - title: string
    - description: string
    - iconData: string


### 消息类型

- 扩展需要发送的消息类型
    - EXTENSION_SUBMIT_WEBSITE_META
- 扩展需要接收和回复的消息类型
    - START_PAGE_REQUEST_WEBSITE_META EXTENSION_RESPONSE_WEBSITE_META
    - START_PAGE_BATCH_REQUEST_METAS EXTENSION_RESPONSE_BATCH_METAS
    - START_PAGE_REQUEST_UNSYNCED_METAS EXTENSION_RESPONSE_UNSYNCED_METAS


### 扩展获取数据的方式

#### 如果是当前打开的标签页
- 通过 chrome.scripting.executeScript 注入脚本直接获取

#### 如果不是当前打开的标签页
- 使用fetch获取数据 
    - fetch(url) 
    - DOMParser()
    - 拿到 title desc iconUrl
    - 使用fetch获取icon数据
    - base64编码
- 如果fetch获取数据失败，在扩展在打开一个标签页获取数据
    - chrome.tabs.create
    - 注入脚本
    - document.xxx 获取 title desc iconUrl
    - fetch(iconUrl)，如果这个有跨域问题，把iconUrl发送给扩展，扩展没有跨域问题
    - base64编码
- icon的备用方案(如果上边方法不行，使用这个方法试试)
    - 这个是之前版本的方法，可能是注入脚本fetch有跨域问题
    - 如果上边使用扩展获取了icon数据，就用不着这个方法了
    - 创建Image对象加载图标
    - 使用Canvas将图标转换为base64
    - 处理跨域问题（设置 crossOrigin = 'anonymous' ）

### 起始页使用扩展获取数据的地方有三个
- 起始页打开的时候(onMounted)，获取未同步的数据
- 起始页添加网站页面，点击获取信息按钮，获取单个网站数据
- 起始页导入数据页面，点击导入按钮，导入数据过程中需要批量获取网站数据
