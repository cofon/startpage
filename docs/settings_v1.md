# 设置面板设计方案 v1.0

## 一、整体布局设计

### 1. 上方按钮行

**按钮列表**（从左到右）：
- 主题 (icon)
- 搜索 (icon)
- 添加网站 (icon)
- 导入 (icon)
- 导出 (icon)
- 搜索结果显示模式切换（Switch）

**布局说明**：
- 所有按钮在一行显示
- 适配最小分辨率：1366×768（14寸笔记本）
- 不考虑手机和平板设备

**按钮行为**：
- 点击"主题"、"搜索"、"添加网站"按钮：在下方显示对应的设置面板
- 点击"搜索结果显示模式切换"按钮：直接切换模式，保存到数据库，关闭设置面板
- 点击"导入"、"导出"按钮：直接执行，显示通知条反馈结果

---

## 二、主题设置面板

### 1. 主题颜色设计

**基础颜色**：
- `primary`：主色调（按钮、链接等）
- `primaryHover`：主色调悬停态
- `primaryActive`：主色调激活态

**文本颜色**：
- `textMain`：主要文本（标题、正文）
- `textSecondary`：次要文本（描述、辅助信息）
- `textDisabled`：禁用文本
- `textOnPrimary`：主色调上的文字（如按钮文字）

**背景颜色**：
- `bgPage`：页面背景
- `bgCard`：卡片背景
- `bgHover`：悬停背景
- `bgActive`：激活背景

**边框和阴影**：
- `borderBase`：基础边框
- `borderFocus`：聚焦边框
- `shadowLight`：浅阴影
- `shadowMedium`：中等阴影
- `shadowDark`：深阴影

### 2. 主题选择列表设计

**视觉设计**：
- 使用圆形色块展示主题的主色调
- 选中的主题：
  - 外圈加粗边框（2-3px）
  - 边框颜色使用当前主题的 `primary` 颜色
  - 添加一个小勾选图标在右上角
- 未选中的主题：
  - 无边框或细边框（1px）
  - 边框颜色使用 `borderBase`

**交互设计**：
- 点击主题色块切换主题
- 切换时添加过渡动画（0.3s）

### 3. 主题颜色编辑

**布局**（从上到下）：
1. 主题选择列表（一行圆形色块）
2. 当前选中主题的可设置颜色列表
3. 主题名称输入框
4. 确认按钮

**颜色列表显示**：
- 每个颜色一行
- 显示内容：颜色名称、颜色方块、颜色值输入框
- 点击颜色方块显示调色板选择颜色
- 颜色值即时更新
- 点击确认按钮修改颜色并更新到数据库

**按钮状态**：
- 默认不可点击
- 修改数据后可点击
- 修改主题名称：按钮文字显示"添加主题"
- 修改主题颜色：按钮文字显示"修改主题"
- 默认显示"添加主题"

### 4. 数据库设计

**主题表（themes）**：
```javascript
{
  id: string, // 主题ID（唯一标识）
  name: string, // 主题名称
  colors: object, // 颜色配置对象
  createdAt: Date,
  updatedAt: Date
}
```

**用户设置表（settings）**：
```javascript
{
  id: string, // 用户ID（单用户应用可以是固定值）
  selectedThemeId: string, // 选中的主题ID
  searchResultLayout: string, // 搜索结果布局模式（'list' | 'grid'）
  selectedSearchEngineId: string, // 选中的搜索引擎ID
  // ...其他用户设置
  updatedAt: Date
}
```

## 三、搜索设置面板

### 1. 搜索引擎列表

**显示内容**：
- 每个搜索引擎一行
- 显示：图标、名称、搜索引擎模板
- 操作按钮：编辑、删除

### 2. 搜索引擎编辑面板

**布局**（从上到下）：
1. 上传图标按钮、图标预览、图标字符串输入框
2. 搜索引擎名称输入框
3. 搜索引擎模板输入框
4. 图标颜色方块和颜色值输入框
5. 确认按钮、取消按钮

**编辑模式判断**：
- 点击列表中的"编辑"按钮：填充数据到编辑区，记录当前编辑的ID
- 点击"添加新搜索引擎"按钮：清空编辑区，ID设为空

**验证规则**：
- 名称：必填，1-50字符
- 模板：必填，必须包含 `{query}` 占位符
- 图标：必填（可以是上传的图片或生成的图标）

### 3. 数据库设计

**搜索引擎表（searchEngines）**：
```javascript
{
  id: string, // 搜索引擎ID（唯一）
  name: string, // 名称
  icon: string, // 图标
  template: string, // 模板
  iconColor: string, // 图标颜色
  order: number, // 排序
  createdAt: Date,
  updatedAt: Date
}
```

## 四、搜索结果显示模式切换

### 1. Switch组件设计

**视觉设计**：
- 容器：100px宽，32px高，圆角16px
- 滑块：60px宽，28px高，圆角14px
- 文字：居中显示，字号12px
- 过渡动画：0.3s ease-in-out

**状态设计**：
- Grid模式：滑块在左侧，显示"G"
- List模式：滑块在右侧，显示"L"

### 2. 交互设计

- 点击切换时：
  - 立即更新UI
  - 保存到数据库
  - 关闭设置面板
  - 显示成功提示

### 3. 数据存储

- 存储位置：用户设置表（settings）
- 字段名：`searchResultLayout`
- 默认值：`'list'`

---

## 五、添加网站设置面板

### 1. 表单字段

**必填字段**：
- 名称：1-100字符
- URL：有效的URL格式

**可选字段**：
- 描述：0-500字符
- 标签：0-10个标签，每个标签1-20字符

**默认值**：
- `isMarked`：false
- `markOrder`：0
- `visitCount`：0
- `isActive`：true
- `isHidden`：false

### 2. 标签输入优化

**标签提示框设计**：
- 显示位置：输入框下方
- 显示内容：现有标签列表（从所有网站的标签中提取）
- 交互：点击标签自动添加到输入框，用逗号分隔
- 样式：标签胶囊样式，可点击

---

## 六、导入导出功能

### 1. 导出功能

**导出格式**：
- JSON格式
- 包含所有网站数据
- 文件名：`startpage_backup_YYYYMMDD_HHMMSS.json`

### 2. 导入功能

**导入流程**：
1. 选择文件
2. 验证JSON格式
3. 显示导入预览（网站数量）
4. 确认后导入
5. 显示导入结果（成功/失败数量）

**冲突处理**：
- ID冲突：使用新ID
- 名称冲突：保留两个（用户后续可手动处理）

---

## 七、用户体验优化

### 1. 设置面板动画

- 打开/关闭：从上到下滑动动画
- 面板切换：淡入淡出动画
- 时长：0.3s

### 2. 数据持久化

- 所有设置更改后立即保存到数据库
- 添加防抖机制，避免频繁写入
- 保存失败时显示错误提示

### 3. 响应式设计

- 设置面板居中显示
- 最大宽度：800px

### 4. 其他优化

- 添加重置按钮（恢复默认设置）
- 添加设置预览（实时预览主题效果）
- 添加键盘快捷键（Esc关闭面板）

## 八、数据存储说明

### 关于 selected search engine 和 selected theme 的存储位置

**说明**：
- `selectedSearchEngineId` 和 `selectedThemeId` 应该保存在 **用户设置表（settings）** 中
- 原因：
  1. 这些是用户偏好设置，不是主题或搜索引擎本身的属性
  2. 将用户选择与数据本身分离，便于管理
  3. 如果需要支持多用户，可以轻松扩展

**存储结构**：
```javascript
// settings 表
{
  id: string, // 用户ID（单用户应用可以是固定值）
  selectedThemeId: string, // 选中的主题ID
  searchResultLayout: string, // 搜索结果布局模式
  selectedSearchEngineId: string, // 选中的搜索引擎ID
  updatedAt: Date
}
```

## 九、注意事项

1. 当前项目的主题只有很少的几个颜色，只是为了测试，需要改写大量代码以支持完整的主题系统
2. 搜索引擎不需要 `isDefault` 字段，因为关闭网页时会保存选中的搜索引擎
3. 搜索结果显示模式切换和网站添加功能需要先看效果才能做出最终决定
4. 所有数据库操作都需要添加错误处理
5. UI设计需要考虑视觉一致性和用户体验
